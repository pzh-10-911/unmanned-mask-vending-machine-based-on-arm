## 二、app.py 修改详情

### 2.1 修复首页 Content-Type 为 text/plain（严重 BUG）

**问题：** 手机浏览器访问首页时 Content-Type 为 `text/plain`，导致浏览器不渲染 HTML。

**根因：** 两个因素叠加——
1. `Flask(__name__, static_url_path='')` 会将 `static/` 目录映射到 URL 根路径 `/`，Werkzeug 的 SharedDataMiddleware 在 `/` 直接返回静态文件，Flask 的 `index()` 路由从未被调用
2. Windows 的 mimetypes 数据库将 `.html` 识别为 `text/plain`

**修改前：**
```python
app = Flask(__name__, static_folder='static', static_url_path='')

@app.route('/')
def index():
    return send_from_directory('static', 'index.html')
```

**修改后：**
```python
app = Flask(__name__, static_folder='static', static_url_path='/static')

@app.route('/')
def index():
    with open('static/index.html', 'r', encoding='utf-8') as f:
        html = f.read()
    return html, 200, {'Content-Type': 'text/html; charset=utf-8'}
```

**关联修改：** `static/index.html` 中 CSS/JS 路径从相对路径改为绝对路径：
```html
<!-- 修改前 -->
<link rel="stylesheet" href="style.css">
<script src="script.js"></script>

<!-- 修改后 -->
<link rel="stylesheet" href="/static/style.css">
<script src="/static/script.js"></script>
```

### 2.2 修复 api_status 返回字段不完整（严重 BUG）

**问题：** 前端需要 `state_name` 和 `selected_channel` 字段，但原代码手动构建 dict，缺少这两个字段。

**修改前：**
```python
@app.route('/api/status')
def api_status():
    return jsonify({
        "state": sm.state if hasattr(sm, 'state') else "IDLE",
        "balance": sm.balance if hasattr(sm, 'balance') else 0,
        "channels": inv.get_status()
    })
```

**修改后：**
```python
@app.route('/api/status')
def api_status():
    return jsonify(sm.get_state())
```

`sm.get_state()` 返回完整结构：
```json
{
    "state": 0,
    "state_name": "IDLE",
    "balance": 0,
    "selected_channel": null,
    "channels": [...]
}
```

### 2.3 修复 api_reset 不重置状态机（中等 BUG）

**问题：** 调用 `/api/reset` 后库存恢复但余额和状态残留，导致连续测试时上轮余额延续。

**修改前：**
```python
@app.route('/api/reset', methods=['POST'])
def api_reset():
    inv.channels = [dict(ch) for ch in inv.DEFAULT_CHANNELS]  # 此处还有 BUG
    inv.transactions = []
    inv.save()
    return jsonify({"success": True})
```

**修改后：**
```python
@app.route('/api/reset', methods=['POST'])
def api_reset():
    inv.channels = []
    for ch in CHANNEL_CONFIG:
        inv.channels.append({
            "id": ch["id"],
            "name": ch["name"],
            "price": ch["price"],
            "stock": ch["init_stock"]
        })
    inv.transactions = []
    inv.save()
    sm.state = 0
    sm.balance = 0
    sm.selected_channel = None
    return jsonify({"success": True})
```

**额外修复：** 原代码 `inv.DEFAULT_CHANNELS` 属性不存在（AttributeError），改为直接从 `CHANNEL_CONFIG` 重建。

### 2.4 修复取消/取消选择后 LED 不熄灭（中等 BUG）

**问题：** 在 DISPENSE 状态下取消交易后，RGB-LED 保持亮起，只能等 30 秒超时线程才熄灭。

**修改前：**
```python
@app.route('/api/cancel', methods=['POST'])
def api_cancel():
    result = sm.cancel()
    return jsonify(result)

@app.route('/api/cancel_select', methods=['POST'])
def api_cancel_select():
    result = sm.cancel_select()
    return jsonify(result)
```

**修改后：**
```python
@app.route('/api/cancel', methods=['POST'])
def api_cancel():
    result = sm.cancel()
    rgb.off()
    buzzer.beep(freq=2000, duration=0.5)  # 新增：高音长鸣退币提示
    log(f"取消交易  |  余额清零  |  退币{sm.balance}元")
    return jsonify(result)

@app.route('/api/cancel_select', methods=['POST'])
def api_cancel_select():
    result = sm.cancel_select()
    rgb.off()
    log(f"取消选择  |  余额保留={sm.balance}")
    return jsonify(result)
```

### 2.5 修复红外取货误触发（功能改进）

**问题：** 手快速划过传感器即触发取货确认，演示时容易误触。

**修改前：**
```python
def wait_for_pickup(channel: int):
    for _ in range(IR_RETRY_COUNT):
        time.sleep(IR_DETECT_INTERVAL)
        if ir.is_blocked():
            sm.complete()
            rgb.off()
            return
    sm.cancel()
    rgb.off()
```

**修改后：**
```python
def wait_for_pickup(channel: int):
    time.sleep(2)                    # 初始延迟，避免手快速划过误触
    blocked_count = 0
    for _ in range(IR_RETRY_COUNT):
        time.sleep(IR_DETECT_INTERVAL)
        if ir.is_blocked():
            blocked_count += 1
            if blocked_count >= 3:   # 连续3次遮挡才确认（1.5秒稳定检测）
                log(f"取货成功  货道{channel}  交易完成")
                sm.complete()
                rgb.off()
                buzzer.beep(freq=1000, duration=0.1)  # 取货成功蜂鸣两声
                time.sleep(0.15)
                buzzer.beep(freq=1000, duration=0.1)
                return
        else:
            blocked_count = 0
    log(f"取货超时  货道{channel}  自动取消")
    sm.cancel()
    rgb.off()
    buzzer.beep(freq=2000, duration=0.5)   # 超时高频长鸣告警
```

### 2.6 新增终端实时操作日志

```

[14:30:02] 投币 +1  |  余额=2
[14:30:05] 选品 成人口罩  ¥2.0  |  余额=2
[14:30:08] 确认购买  货道0  |  余额=0.0
[14:30:13] 取货成功  货道0  交易完成
```

实现方式：在文件顶部定义 `log()` 函数，所有 API 端点调用时输出：
```python
from datetime import datetime

def log(msg):
    ts = datetime.now().strftime("%H:%M:%S")
    print(f"[{ts}] {msg}", flush=True)
```

### 2.7 新增强制禁用浏览器缓存

**问题：** 修改前端文件后手机浏览器仍加载旧版缓存。

```python
@app.after_request
def add_no_cache(response):
    response.headers['Cache-Control'] = 'no-cache, no-store, must-revalidate'
    response.headers['Pragma'] = 'no-cache'
    response.headers['Expires'] = '0'
    return response
```

## 三、logic/state_machine.py 修改详情

### 3.1 cancel() 允许在 IDLE 状态退币

**问题：** 用户投币后未选品时点击"取消交易"，后端返回"当前状态不允许取消"，余额无法清零。

**修改前：**
```python
def cancel(self) -> dict:
    if self.state not in (SELECTED, DISPENSE):
        return {"success": False, "msg": "当前状态不允许取消"}
    self.balance = 0
    self.state = IDLE
    self.selected_channel = None
    return {"success": True, "balance": 0}
```

**修改后：**
```python
def cancel(self) -> dict:
    self.balance = 0
    self.state = IDLE
    self.selected_channel = None
    return {"success": True, "balance": 0}
```

## 四、static/index.html 修改详情

### 4.1 整体架构变更

**修改前：** 三页面切换模式（page-main / page-confirm / page-result），通过 `showPage()` 切换 display

**修改后：** 弹出框模式，确认弹窗和结果弹窗使用 `position:fixed` 遮罩层，不切换页面。优势：
- 避免页面跳转导致的事件冲突
- 取货完成后前端轮询检测 DISPENSE→IDLE 自动关闭结果弹窗
- 操作更流畅，无页面闪烁

### 4.3 结果弹窗自动关闭逻辑

前端每 1.5 秒轮询 `/api/status`，记录上一次的 state 值。当检测到 `state` 从 `2`（DISPENSE）变为 `0`（IDLE）时，判定取货完成，自动关闭结果弹窗：

```javascript
function fetchStatus(){
    api('/api/status').then(function(d){
        var prev=_state.state;
        _state=d;
        // ... 更新 UI ...
        if(prev===2 && d.state===0){
            document.getElementById('popup-result').style.display='none';
            if(_timer){clearInterval(_timer);_timer=null;}
        }
    }).catch(function(){});
}
```
